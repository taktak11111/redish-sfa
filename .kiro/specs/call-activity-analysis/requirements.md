# 要件定義書: 架電活動分析機能 (Call Activity Analysis)

## 1. はじめに
インサイドセールス（IS）の活動量と質を可視化し、アポイント獲得の効率化と商談の有効率向上を支援する。

## 2. ユーザーストーリー
- **ISリーダーとして**: チーム全体の活動量（架電数）と結果（アポ数）を把握し、リソースの最適化を行いたい。
- **経営者として**: リードソース（チャネル）ごとのアポ獲得コストと効率を比較し、マーケティング投資の判断材料にしたい。
- **FS（フィールドセールス）として**: ISが獲得したアポがどれだけ実際に商談に繋がっているか（実施率）を確認し、連携を強化したい。

## 3. 主要KPI定義
1. **総架電数**: 指定期間内の全架電試行回数。
2. **商談獲得数（アポ数）**: ステータスが「09.アポ獲得」または「03.アポイント獲得済」になった件数。
3. **商談獲得率 (Conversion Rate)**: アポ数 ÷ 総架電数。
4. **商談実施率 (Setup Rate)**: 商談が設定・実施された数 ÷ アポ数。

## 4. 分析軸 (Analysis Dimensions)
- **担当者別**: 個別のパフォーマンス比較。
- **チャネル別 (Lead Source)**: Meetsmore, TEMPOS, Amazon 等。
- **商材（サービス）別**: RO:開業, RT:税務 等。
- **期間別**: 日次・週次・月次のトレンド推移。

## 5. UI要件
- **サマリーカード**: 上記4つの主要KPIを大きく表示。
- **活動推移グラフ**: 架電数とアポ数を並べたトレンドチャート。
- **多軸比較テーブル**: 担当者/チャネル/商材を切り替えて、架電数・アポ数・各率を一覧表示。

## 6. 技術仕様
- **URL**: `/analysis/calls`
- **API**: `GET /api/analysis/calls`
- **データソース**: `call_records`, `deals`
