# 不整合調査レポート：詳細画面と設定メニューの比較

**調査日**: 2026-01-13  
**調査対象**: 架電管理、商談管理、成約・契約管理の詳細画面と設定メニュー

## 調査概要

## 4. 不整合の重要度分類

### 🔴 重大な不整合（即座に対応が必要）

1. **商談実施状況**: 設定メニューの`meetingStatus`と詳細画面の選択肢が不一致
   - 設定メニュー: `商談キャンセル`, `ノーショー（連絡なし）`, `商談実施`, `商談再設定`
   - 詳細画面: `実施前`, `実施済`, `ノーショー 連絡なし`, `キャンセル`, `リスケ`

2. **商談最終結果**: 設定メニューの`dealResult`と詳細画面の選択肢が不一致
   - 設定メニュー: `成約`, `検討中`, `失注`
   - 詳細画面: `成約（即決）`, `成約`, `失注`

3. **成約要因**: 設定メニューに存在しない（新規追加が必要）

4. **ISへのフィードバック**: 設定メニューに存在しない（新規追加が必要）

5. **ニーズ温度（IS判定）**: 架電管理・商談管理の両方で使用されているが、設定メニューに存在しない（新規追加が必要）

### 🟡 部分的不整合（要確認・調整）

1. **BANT情報（B:予算, A:決裁権）**: 設定メニューの`bantInfo`と詳細画面の選択肢の形式が異なる

2. **T:導入時期**: 設定メニューの`openingPeriod`と詳細画面の選択肢が不一致

3. **競合・自己対応状況**: 設定メニューの`competitorStatus`と詳細画面の選択肢が不一致

4. **商談フェーズ**: 設定メニューは4択だが、詳細画面では2択のみ

5. **次回アクション内容**: 詳細画面で`回答確認`が追加されているが、設定メニューに存在しない

### ⚠️ 軽微な不整合（固定値として扱う場合は問題なし）

1. **成約・契約管理の各種選択肢**: 固定値として扱う場合は問題なし

## 5. 推奨対応方針

### 優先度1（即座に対応）

1. **設定メニューに新規項目を追加**:
   - `needTemperature`: ニーズ温度（IS判定）（A/B/C）
   - `contractReason`: 成約要因
   - `feedbackToIS`: ISへのフィードバック

2. **既存項目の選択肢を統一**:
   - `meetingStatus`: 詳細画面の選択肢に合わせて設定メニューを更新、または詳細画面を設定メニューに合わせる
   - `dealResult`: `成約（即決）`を追加、または詳細画面から削除

### 優先度2（要確認・調整）

1. **BANT情報の形式統一**: 設定メニューの`bantInfo`を詳細画面の形式に合わせる、または詳細画面を設定メニューに合わせる

2. **導入時期の選択肢統一**: 設定メニューの`openingPeriod`と詳細画面の選択肢を統一

3. **競合・自己対応状況の選択肢統一**: 設定メニューの`competitorStatus`と詳細画面の選択肢を統一

4. **商談フェーズの選択肢統一**: 詳細画面で使用する選択肢を明確化（2択で問題ない場合は設定メニューから削除、または詳細画面で4択を使用）

5. **次回アクション内容**: 設定メニューに`回答確認`を追加

### 優先度3（将来対応）

1. **成約・契約管理の選択肢**: 必要に応じて設定メニューで管理
