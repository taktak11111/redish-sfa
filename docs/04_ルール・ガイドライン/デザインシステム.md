# REDISH SFA デザインシステム

> **最終更新**: 2024年12月7日  
> **ベース**: REDISH開業システム デザインシステム

---

## 概要

REDISH SFAは、REDISH開業システムと同一のブランドカラー・デザイン原則を共有しながら、営業支援システム固有の要件に対応したデザインシステムを採用しています。

---

## 1. カラーシステム

### 1.1 ブランドカラー（プライマリー）

REDISHのコーポレートカラーであるターコイズブルーを採用。

```css
primary-50:  #e6f7fa  /* 背景・ホバー */
primary-100: #b3e8f0  /* 淡い背景 */
primary-200: #80d9e6  /* ボーダー */
primary-300: #4dcadc  /* アイコン */
primary-400: #1ab8d2  /* ボタン（ホバー前） */
primary-500: #00a4c5  /* メインカラー */
primary-600: #0083a0  /* ボタン・アクション */
primary-700: #00627b  /* テキスト（強調）※コントラスト比6.5:1 */
primary-800: #004156  /* テキスト（濃い） */
primary-900: #002031  /* 最濃色 */
```

### 1.2 ニュートラルカラー

```css
base:  #FFFFFF  /* 背景 */
sub1:  #231815  /* メインテキスト */
sub2:  #E2E2E3  /* ボーダー・区切り線 */
```

### 1.3 セマンティックカラー

```css
/* 成功・完了 */
success-light:   #D1FAE5
success:         #10B981
success-dark:    #065F46

/* 警告・注意 */
warning-light:   #FEF3C7
warning:         #F59E0B
warning-dark:    #92400E

/* エラー・危険 */
error-light:     #FEE2E2
error:           #EF4444
error-dark:      #991B1B

/* 情報 */
info-light:      #DBEAFE
info:            #3B82F6
info-dark:       #1E40AF
```

### 1.4 SFA専用カラー

#### 確度ランク

営業活動の確度を視覚的に表現します。

| ランク | カラー | 用途 |
|--------|--------|------|
| A:80% | 緑（success系） | 高確度 |
| B:50% | 青（info系） | 中確度 |
| C:20% | 黄（warning系） | 低確度 |
| D:10% | 赤（error系） | 要注意 |

```css
/* 確度ランク */
rank-a: #10B981  /* 緑 */
rank-b: #3B82F6  /* 青 */
rank-c: #F59E0B  /* 黄 */
rank-d: #EF4444  /* 赤 */
```

#### 架電ステータス

```css
call-uncalled:   #6B7280  /* 未架電（グレー） */
call-calling:    #3B82F6  /* 架電中（青） */
call-appointed:  #10B981  /* アポ獲得（緑） */
call-noappoint:  #EF4444  /* アポなし（赤） */
```

#### 商談結果

```css
deal-active:     #3B82F6  /* 進行中（青） */
deal-contract:   #10B981  /* 成約（緑） */
deal-lost:       #EF4444  /* 失注（赤） */
```

## 2. カラー使用ルール

### 2.1 必須ルール

| ルール | 内容 |
|--------|------|
| ✅ 推奨 | Tailwindのカスタムカラー（`primary-500`等）を使用 |
| ❌ 禁止 | ハードコード（`#00a4c5`等）は禁止 |
| ⚠️ 注意 | コントラスト比を考慮（テキストは`primary-700`以上） |

### 2.2 用途別カラー

| 用途 | 推奨クラス |
|------|-----------|
| メインアクション | `bg-primary-600 hover:bg-primary-700` |
| セカンダリアクション | `border-primary-200 text-primary-700 bg-white` |
| テキストリンク | `text-primary-700`（コントラスト比6.5:1） |
| 成功・完了 | `bg-success-light text-success-dark` |
| 警告・注意 | `bg-warning-light text-warning-dark` |
| エラー | `bg-error-light text-error-dark` |
| 情報 | `bg-info-light text-info-dark` |

## 5. レスポンシブデザイン

### ブレークポイント

| プレフィックス | 最小幅 | 用途 |
|---------------|--------|------|
| (なし) | 0px | モバイル |
| `sm:` | 640px | 小型タブレット |
| `md:` | 768px | タブレット |
| `lg:` | 1024px | デスクトップ |
| `xl:` | 1280px | 大画面 |

### グリッドの例

```tsx
/* レスポンシブグリッド */
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4">

</div>
```

## 6. チェックリスト

新しいUIを実装する前に確認：

- [ ] カスタムカラー（primary-500等）を使用しているか
- [ ] ハードコードされた色がないか
- [ ] テキストのコントラスト比は十分か（primary-700以上）
- [ ] 統一されたコンポーネントスタイルを使用しているか
- [ ] レスポンシブ対応しているか
- [ ] 確度・ステータスには適切なバッジを使用しているか
