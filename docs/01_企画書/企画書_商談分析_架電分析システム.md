## 1. エグゼクティブサマリー

### 1.1 プロジェクト概要

本システムは、商談・架電のテキストデータをAI分析し、**営業担当者の成長を促す学習プラットフォーム**です。

**REDISHナレッジプラットフォーム構想**の「無形資産」版として、営業組織の知識・スキル・マインドセットを資産化します。

### 1.2 核心コンセプト

```
┌─────────────────────────────────────────────────────────────────┐
│             解決すべき本質的課題（営業版）                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   「自分の何が良くて何が悪いのかわからない」                    │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  営業担当者は客観的な自己評価が難しい                    │   │
│   │  → 課題発見から改善まで一貫してサポートが必要            │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   【解決アプローチ】                                             │
│   ① 課題発見: AI分析で「あなたの改善ポイント」を明確化         │
│   ② 学習促進: 必要なスキル・知識を段階的に提供                 │
│   ③ 実践支援: チェックリストで次の商談に活かす                 │
│   ④ 成長実感: バッジ・スコア推移で成長を可視化                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 提供価値

| 対象者 | 提供価値 |
|--------|----------|
| **営業担当者** | 客観的フィードバック → 課題発見 → 学習 → 実践の好循環 |
| **営業マネージャー** | データに基づくコーチング、組織課題の可視化 |
| **経営層** | 営業組織の無形資産化、ナレッジの蓄積・継承 |

## 2. 設計思想：ナレッジプラットフォームの応用

### 2.1 開業システムからの学び

**REDISHナレッジプラットフォーム構想**の核心は：
- 「わからないことがわからない」状態からの脱却
- 課題発見 → 学習 → 実践 → 成長の好循環
- ナレッジの資産化と組織全体への展開

これを営業組織に適用します。

### 2.2 営業版ナレッジプラットフォーム

```
┌─────────────────────────────────────────────────────────────────┐
│               営業版ナレッジプラットフォームの好循環              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   【Phase 1: 課題発見】                                          │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  担当者: 「何が良くて何が悪いのかわからない」            │   │
│   │                    ↓                                     │   │
│   │  AI分析: 商談/架電テキストを構造的にタグ付け・評価       │   │
│   │                    ↓                                     │   │
```

## 3. 営業コンピテンシーフレームワーク

### 3.1 REDISH独自のセールス定義

```
┌─────────────────────────────────────────────────────────────────┐
│                    REDISHセールス哲学                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【セールス定義】                                                │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  セールス ＝ 課題解決                                     │   │
│  │  売ることは課題解決の一部に過ぎない                       │   │
│  │                                                           │   │
│  │  本質: WIN-WIN                                            │   │
│  │  ・必要条件: 自分の利益のために売りつける → 売れない      │   │
│  │  ・十分条件: 顧客のために課題解決できれば → 売れる        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  【What: 何を売っているか】                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  無形商材 ＝「経営」という商材                            │   │
│  │  ├─ ファイナンス（お金周り）                             │   │
│  │  └─ マーケティング（集客）                               │   │
│  │                                                           │   │
│  │  ★ 課題認識・課題の顕在化が最も重要                       │   │
│  │    （顧客は具体的欲求や課題があまりない状態）             │   │
│  │  ★ ファイナンスとマーケティングに課題認識を収斂させる    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  【Who: 誰に売っているか・その課題は】                          │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  ・「わからないことがわからない」経営がわからない         │   │
│  │  ・面倒・苦手意識                                         │   │
│  │                                                           │   │
│  │  参考：顕在課題（現象）                                   │   │
│  │  ├─ 集客・人材・資金                                     │   │
│  │  └─ 現象:課題 = 結果とプロセス・原因 = 具体と抽象        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  【How: どうやって課題解決するか】                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  エトス・パトス・ロゴス（信頼・共感・ロジック）           │   │
│  │                                                           │   │
│  │  ① あなたに依頼したい: エトス・パトス                    │   │
│  │  ② どうやって課題解決するか: ロゴス                      │   │
│  │     ・価値＞価格（自社商材の価値）                        │   │
│  │     ・体感（課題解決後のBefore/After）                    │   │
│  │     ・競合比較（他社・自己処理との比較）                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
